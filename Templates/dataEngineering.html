Data Transformation Process

Main preprocessing and feature engineering techniques applied before model training

Temporal Separation

  • The period variable was divided into two new columns: year and season.
  • This transformation improves temporal interpretation and allows the model to identify seasonal climate behavior associated with pest propagation.

Missing Values Handling

  • Missing values were detected in the pest_name and species columns.
  • To avoid inconsistencies during model training, empty values were replaced with the labels Pest_free and Unknown.

Feature Engineering

  • A new variable called climate_risk was created based on temperature and precipitation thresholds.
  • If the average temperature exceeded 28°C and precipitation was greater than 100 mm, the record was classified as high climate risk.
  • This variable helps represent environmental conditions favorable to pest propagation.

Data Categorization

  • Temperature and precipitation values were grouped into low, medium, and high categories.
  • This transformation simplifies climate interpretation and improves pattern detection for the machine learning model.

Encoding Process

  • Machine learning algorithms cannot process text values directly.
  • Categorical variables such as municipality, crop type, species, and pest names were transformed into numerical representations using Label Encoding.

Normalization

  • Numerical variables such as crop yield, relative yield, temperature, and precipitation were normalized using MinMaxScaler.
  • This process scales values between 0 and 1, preventing variables with large ranges from dominating the training process.
