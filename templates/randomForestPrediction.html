Business Understanding

This system supports agricultural decision-making by helping identify pest risk conditions before outbreaks escalate.

Problem to Solve

Farmers and agronomists need a practical way to estimate pest risk from environmental and production variables without waiting for manual inspection or delayed field reports.

Expected Impact

The application reduces uncertainty, improves preventive action, and supports faster responses when climate conditions suggest a high risk of infestation.

Users of the System

The interface is designed for agricultural technicians, researchers, and decision-makers who need a simple tool to classify new cases using model predictions.

Practical Goal

The main goal is to convert a trained machine learning model into a usable web tool with an intuitive interface and clear prediction output.

Data Understanding

The model relies on structured agricultural and climate variables that describe local conditions before prediction.

Main Inputs

The prediction form captures features such as average temperature, total precipitation, relative yield, and municipality to build the input vector required by the Random Forest model.

Feature Role

Temperature and precipitation provide climate context, while location and yield-related variables add agricultural structure to the classification process.

Data Preparation

Before reaching this interface, the data should already have been cleaned, encoded, and prepared using the preprocessing steps applied during model engineering.

Input Validation

The form expects numeric values where needed and helps avoid invalid requests that could break the prediction workflow in the backend.

Model Engineering

The Random Forest model combines multiple decision trees to improve stability and reduce overfitting.

Why Random Forest

Random Forest is useful for tabular data because it handles non-linear relationships well and usually performs better than a single tree on noisy real-world datasets.

Prediction Behavior

The model aggregates the results of many trees to produce a final class, which makes the classification more robust than using one isolated rule path.

Probability Output

If the model supports predict_proba, the application can also show confidence percentages, which makes the result easier to interpret for end users.

Backend Role

The Flask route receives the values from the form, formats them as features, sends them to the model, and returns the prediction to the template.

Model Evaluation

Evaluation explains how well the model behaves and why its output can be trusted for practical use.

Classification Output

The system can return class labels such as Alert or No Alert, depending on the target encoding used during training.

Confidence Measure

When probabilities are available, the model can show how confident it is in the predicted class, which helps users interpret uncertain predictions.

Error Handling

If the backend receives incomplete or invalid values, the app can catch the exception and display a user-friendly message instead of crashing.

Deployment Value

This evaluation stage connects the analytical model with the final user experience, turning a trained algorithm into an operational decision-support tool.

Manual Prediction Input

Enter the required variables manually to obtain a pest risk classification from the Random Forest model.

Input Requirements

This interface allows users to introduce data manually, submit a prediction request, and receive a classification result directly in the browser.

Backend Workflow

The Flask backend receives the form data, prepares the feature vector, calls the trained model, and returns the predicted class and confidence score.

Error Handling

If any value is invalid or missing, the system can display a clear message so the user knows what needs to be corrected before retrying.

Output Display

The prediction result can be shown as a label such as Alert or No Alert, along with the probability of the predicted class.
