Dataset Assessment

Why this dataset is useful for the project

📌

Relevance

  • The dataset contains agricultural, environmental, and crop-related variables connected to pest propagation.
  • Its structure supports classification, preprocessing, and exploratory analysis.
  • The information is aligned with the project objective and can be used to build a useful academic model.
🔎

Data quality

  • The data is organized in tabular form, which makes cleaning and transformation easier.
  • Missing values, duplicates, and inconsistent categories should be reviewed before training the model.
  • Numeric and categorical variables can both contribute to the analysis after preprocessing.

Analysis of Viability

Technical, operational, and data feasibility

01

Technical viability

The dataset can be processed with Python, pandas, NumPy, and scikit-learn. This makes it suitable for cleaning, encoding, scaling, and model training.

  • Python supports preprocessing and machine learning.
  • Scikit-learn can train and evaluate predictive models.
  • The data can be transformed before analysis.
02

Operational viability

The results can be shown inside a Flask web app, allowing a simple interface for analysis and presentation of findings.

  • Useful for showing results to the professor.
  • Easy to integrate with navigation and sections.
  • Improves the presentation of the project.
03

Data viability

The dataset appears relevant enough to support a predictive approach, but it must be validated to confirm completeness, consistency, and usefulness.

  • Review missing values.
  • Check duplicate records.
  • Validate categorical labels.
04

Risk management

Some issues can affect the model, such as noisy features, imbalance, or incomplete data. These risks should be handled before training.

  • Identify low-quality variables.
  • Balance classes if necessary.
  • Apply preprocessing before modeling.

Risks and Limitations

Possible issues that may affect the model

Issue Impact Status
Missing values Can reduce model accuracy and require imputation or row removal. Needs review
Duplicate rows May bias the analysis and distort patterns. Check cleaning
Class imbalance Could make some pest categories harder to predict. Potential risk
Noisy features Unimportant variables may lower the model’s performance. Feature selection

Recommended Preprocessing

Steps to improve the dataset before modeling

01

Clean missing values

Identify empty fields and decide whether to impute them or remove incomplete records.

02

Normalize categories

Standardize labels in categorical columns to avoid duplicates caused by spelling differences.

03

Select relevant features

Keep the variables with the highest predictive value and remove unnecessary fields.
